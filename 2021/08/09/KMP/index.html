KMP入门

发表于 2021-08-09 更新于 2021-08-24
本文字数： 4.6k 阅读时长 ≈ 4 分钟

KMP算法是一种字符串匹配算法，可以在 O(n+m) 的时间复杂度内实现两个字符串的匹配。

字符串匹配问题

所谓字符串匹配， 就是假设有两个字符串 s 和 p，问字符串 p 是否是串 s 的子串，如果是子串就输出子串开始的下标，不是的话就输出-1。

这种问题用暴力的做法也很好求， 只要从主串 s 的开头开始逐字符枚举就行了，之后我们只需要比较 i 指针和 j 指针指向的字符是否一致。如果一致就向后移动， 不然就让子串整体右移一位。

上述方法是没有问题的，但时间复杂度是O(n * m)，复杂度一下上天。

在上述方法中，如果从 S[i] 开始的那一趟比较失败了，算法会直接开始尝试从 S[i + 1] 开始比较，我们可以看出，如果 S[i] 与 P 的匹配是在第 r 个位置失败的，那么从 S[i] 开始的 (r-1) 个连续字符，一定与 P 的前 (r-1) 个字符一模一样。

掌握了这些信息后，我们就要根据这些信息推断出，哪些匹配是毫无可能的，并跳过这些匹配。

那么，哪些字符串比较是不可能成功的？来看一个例子。

模式串 P = “abcabd” 和主串从S[0]开始匹配时，在 P[5] 处失配。既然是在 P[5] 失配的，那么说明 S[0 ~ 5] 等于 P[0 ~ 5]，即”abcab”. 现在我们来考虑：从 S[1]、S[2]、S[3] 开始的匹配尝试，有没有可能成功？　　

从 S[1] 开始肯定没办法成功，因为 S[1] = P[1] = ‘b’，和 P[0] 并不相等。从 S[2] 开始也是没戏的，因为 S[2] = P[2] = ‘c’，并不等于P[0]. 但是从 S[3] 开始是有可能成功的（至少按照目前的信息， 我们推不出矛盾的地方。

因此我们最初的找字符串问题的目标开始转向跳过不可能成功的匹配位置， 由此我们引入一个新概念——Next数组。

next 数组

next数组是相对于模式串（子串）而言的，next数组是模式串最长相同的前后缀长度，而且长度必须小于本身，因为等于自己没有意义。next[i] 表示p[0] ~ p[i] 这一个 p 的子串前 k 个字符恰好等于后 k 个字符的最大的数。

比如模式串p = “abcabd”;

i = 0 时，因为子串只有 “a” 所以 长度为0 –> next[0] = 0;

i = 1 时，子串”ab”没有最长相同前后缀 长度为0 –> next[1] = 0;

i = 2 时，”abc” 没有最长相同前后缀 长度为0 –> next[2] = 0;

i = 3 时，”abca” 最长相同前后缀为”a” 长度为1 –> next[3] = 1;

i = 4 时，”abcab”最长相同前后缀为”ab”长度为2–> next[4] = 2;

i = 5 时，”abcabd”没有最长相同前后缀 长度为0–> next[5] = 0;

上图给出了一个例子。P[0 ~ 4]=”abcab”时，next[4]=2，这是因为P[0 ~ 4] 这个子串是”abcab”，前两个字符与后两个字符相等，因此next[4]取2。 而next[5]=0，是因为”abcabd”找不到前缀与后缀相同的地方，因此只能取0。

由此，next数组的作用就显现出来了，主要就是每次失配之后跳过不可能匹配成功的位置，但是我们该如何移位呢？

我们拿s = “ababaabaabac…”, p = “abaabac”来看一下

i = j = 0 时开始匹配，在i = j = 3 失配之后，我们直接将模式串右移了两位，让s[2] = p[0] 继续匹配，最后在i = 8, j = 6时失配，再让模式串右移3位，此后继续匹配直到成功。

即：p[3] 失配后，把 p[next[3-1]] (next[2] = 1) 也就是 P[1] 对准了主串刚刚失配的那一位； P[6] 失配后，把 P[next[6-1]] (next[5] = 3) 也就是P[3]对准了主串刚刚失配的那一位。

如图中蓝色箭头所示，旧的后缀要与新的前缀一致（如果不一致，那就肯定没法匹配上了）

那么，如何来求next数组呢？

求next数组

首先说一句：快速构建next数组，是KMP算法的精髓所在，核心思想是“P自己与自己做匹配”。
　　为什么这样说呢？回顾next数组的完整定义：

  • 定义 “k-前缀” 为一个字符串的前 k 个字符； “k-后缀” 为一个字符串的后 k 个字符。k 必须小于字符串长度。
  • next[x] 定义为： P[0] ~ P[x] 这一段字符串，使得k-前缀恰等于k-后缀的最大的k.

　　这个定义中，不知不觉地就包含了一个匹配——前缀和后缀相等。接下来，我们考虑采用递推的方式求出next数组。如果next[0], next[1], … next[x-1]均已知，那么如何求出 next[x] 呢？

​ 来分情况讨论。首先，已经知道了 next[x-1]（以下记为now），如果 P[x] 与 P[now] 一样，那最长相等前后缀的长度就可以扩展一位，很明显 next[x] = now + 1. 图示如下。

p[x] == p[now] 的情况解决了，那么p[x] != p[now]的情况怎么办呢？

如图。长度为 now 的子串 A 和子串 B 是 P[0] ~ P[x-1] 中最长的公共前后缀。可惜 A 右边的字符和 B 右边的那个字符不相等，next[x]不能改成 now+1 了。因此，我们应该缩短这个now，把它改成小一点的值，再来试试 P[x] 是否等于 P[now].
　　now该缩小到多少呢？显然，我们不想让now缩小太多。因此我们决定，在保持“P[0] ~ P[x-1]的now-前缀仍然等于now-后缀”的前提下，让这个新的now尽可能大一点。 P[0] ~ P[x-1] 的公共前后缀，前缀一定落在串A里面、后缀一定落在串B里面。换句话讲：接下来now应该改成：使得 A的k-前缀等于B的k-后缀 的最大的k.
　　我们应该已经注意到了一个非常强的性质——串A和串B是相同的！B的后缀等于A的后缀！因此，使得A的k-前缀等于B的k-后缀的最大的k，其实就是串A的最长公共前后缀的长度 —— next[now-1]！

所以现在求next数组的方法其实已经很明了了。

p[now] == p[x] 的情况下，next[x] = now + 1;

p[now] != p[x] 的情况下， now = next[now];

字符串匹配

既然next数组构造好了，如何用它来匹配字符串就变成了我们仅剩的一个问题。

又回到了这个图上。 从图中不难看出如果把模式串视为一把标尺，在主串上移动，那么暴力方法就是每次失配之后只右移一位；改进后的算法则是每次失配之后，移很多位，跳过那些不可能匹配成功的位置。

现在来回忆一下next数组的性质：P[0] 到 P[i] 这一段子串中，前next[i]个字符与后next[i]个字符一模一样。既然如此，如果失配在 P[r], 那么P[0] ~ P[r-1]这一段里面，前next[r-1]个字符恰好和后next[r-1]个字符相等——也就是说，我们可以拿长度为 next[r-1] 的那一段前缀，来顶替当前后缀的位置，让匹配继续。

如上图所示，绿色部分是成功匹配，失配于红色部分。深绿色手绘线条标出了相等的前缀和后缀，其长度为next[右端]. 由于手绘线条部分的字符是一样的，所以直接把前面那条移到后面那条的位置。因此说，next数组为我们如何移动标尺提供了依据。
