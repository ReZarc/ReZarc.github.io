线段树入门

发表于 2021-08-31 更新于 2021-09-04
本文字数： 7.2k 阅读时长 ≈ 7 分钟

线段树是一种二叉搜索树，他将一段区间用二分的思想划分为若干单位区间， 每一个结点都存储着一个区间。树状数组可以进行的操作他都可以操作，树状数组不能进行的操作他也可以操作。但是比树状数组难写很多。。。一不小心就能调出来段错误，调个半小时是很正常的事。

基本思想

我们先来看一下这张动图：

线段树的基本思想和二分的思想很像：线段树每一个节点都存储着一段区间 [l, r] 的信息，其中叶子结点上的 l == r。大致思想是：将一段大区间平均地划分成两个小区间，每一个区间再平均划分成两个更小的区间，直到划分到叶子结点。通过这些节点来进行修改、查询。

线段树主要是把一段大区间平均地划分成两段小区间进行维护，再用小区间的值来更新大区间。这样既能保证正确性，又能使时间保持在log级别（因为这棵线段树是平衡的）。也就是说，一个[l, r]的区间内，用mid 代表l + r >> 1，这一区间会被划分成[l, mid]和[mid + 1, r]这两个小区间进行维护，直到 I == r。
上图就是一棵[1,10]的线段树的分解过程（相同颜色的节点在同一层）

通常用的都是堆式储存法，即编号为k的节点的左儿子编号为 i << 1 (2 * i)，右儿子编号为i << 1 | 1 (2 * i + 1)，父节点编号为i >> 1。

通常，每一个线段树上的节点储存的都是这几个变量：区间左边界，区间右边界，区间的答案（这里为区间元素之和）

单点修改

经过上述操作，线段树就建好了，但是如何去修改呢？

假如我们要让一个节点更新它的值，那样会起到牵一发而动全身的效应，因为这个点的值修改后，维护的区间和就改变了，我们应该继续进行pushup操作来进行维护。

例如：a[9] = {0, 4, 6, 8, 9, 3, 1, 4, 7};

我们让a[2] += 9;

所以我们需要对每一个节点更新，使用的数据有本次的根节点，要控制的总范围区间（左，右），要修改的点和要修改的数据。持续更新它的父节点直到顶端即可。

其实如果会了单点修改的话， 区间修改就不难了。

区间修改大致分为两步：
1.找到区间中全部都是需要修改的点的区间

​ 2.修改这一区间中的所有点

先来解决第一步：

我们先从根节点出发，一直左右递归寻找区间，直到当前区间的元素全部都是需要修改的元素。

那么怎么修改呢？一个一个点修改？那样的时间复杂度显然比普通暴力做法还要慢，现在就到了用pushdown的时候了，我们引入一个新东西——懒标记。

懒标记

懒标记的原理在于修改区间的时候在不使用他的子孙时先把这一部分数据扣住，对父亲产生影响并产生一个懒标记。有了这个关键的方法，我们可以建立一个更新的pushdown函数用来下传懒标记和数据。修改时只需要修改区间和就可以了。

区间查询

上面的两个操作我们学会以后，我们就是在维护一个线段树了。但是光维护还是不够的，我们怎么来查询呢？

其实区间查询和单点查询都是差不多的，区间查询是查询区间和，单点查询是查询单点和（听君一席话如听一席话），区间查询得操作和区间修改的操作差不多，都是从根节点左右递归寻找全部包含的区间，然后加起来返回。

例题：CodeForces 242E

题意：给定 n 个数的数列a[], 有 m 次操作，两种操作分别为区间求和与区间异或。每次求和输出。

一般情况下区间求和与区间异或不能同时进行，所以我们将其二进制拆位来维护。
